## Plan: Fix Recent Messages Noise + Consistent Page Width

### Issue 1: Filter out internal MQTT messages from Recent Messages

**Root cause:** The backend publishes `groundcontrol/nfc/config` as a retained message on MQTT connect. Since it subscribes to `#`, the broker echoes it back and it gets stored in the database. The `should_store_message()` function doesn't exclude this topic.

**Fix:** Add an exclusion for `groundcontrol/nfc/config` in `should_store_message()` in `backend/core/mqtt.py:53-67`.

```python
# Add after the existing filters (line ~66):
if topic.startswith("groundcontrol/"):
    return False
```

This filters out all `groundcontrol/` prefixed topics (currently only `groundcontrol/nfc/config` exists), preventing them from being stored and shown in Recent Messages. This matches the existing pattern for other internal/system topics like `zigbee2mqtt`.

**One file changed:** `backend/core/mqtt.py`
